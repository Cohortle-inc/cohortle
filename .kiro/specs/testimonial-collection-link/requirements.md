# Requirements Document

## Introduction

Conveners on Cohortle can currently add testimonials manually through the settings page. This feature extends that capability by allowing a convener to generate a shareable collection link scoped to a specific cohort. Only enrolled participants of that cohort can follow the link and submit a testimonial/review. Submitted testimonials feed into the existing testimonials system and can be moderated or auto-approved based on convener preference.

## Glossary

- **Convener**: A user with the `convener` role who creates and manages programmes and cohorts.
- **Cohort**: A time-bounded group of learners enrolled in a specific programme.
- **Collection_Link**: A unique, token-based URL generated by a convener for a specific cohort that allows enrolled learners to submit testimonials.
- **Collection_Token**: A cryptographically random, URL-safe string that identifies a Collection_Link and scopes it to a cohort.
- **Learner**: A user with the `student` role who is enrolled in a cohort via the `enrollments` table.
- **Testimonial**: A learner review record stored in the `testimonials` table, containing a quote, rating, learner name, and programme name.
- **Submission_Form**: The public-facing page a learner visits via a Collection_Link to submit a testimonial.
- **Auto_Approve**: A convener setting that, when enabled, marks submitted testimonials as `is_featured = true` immediately upon submission.

## Requirements

### Requirement 1: Generate a Collection Link

**User Story:** As a convener, I want to generate a shareable testimonial collection link for a specific cohort, so that I can invite enrolled learners to submit reviews without manually entering each one.

#### Acceptance Criteria

1. WHEN a convener requests a collection link for a cohort they own, THE Collection_Link_Service SHALL generate a unique Collection_Token and persist it with the cohort_id, convener user_id, an optional expiry date, and an auto_approve flag.
2. WHEN a convener requests a collection link for a cohort they do not own, THE API SHALL return a 403 Forbidden response.
3. WHEN a collection link already exists for a cohort, THE Collection_Link_Service SHALL return the existing link rather than creating a duplicate.
4. THE Collection_Link_Service SHALL construct the shareable URL in the format `{BASE_URL}/testimonial/{token}`.
5. WHEN a convener sets an expiry date on a collection link, THE Collection_Link_Service SHALL store the expiry date and enforce it during submission validation.
6. WHEN a convener regenerates a collection link for a cohort, THE Collection_Link_Service SHALL invalidate the previous token and issue a new one.

### Requirement 2: Manage Collection Links

**User Story:** As a convener, I want to view, configure, and revoke collection links for my cohorts, so that I have full control over who can submit testimonials and when.

#### Acceptance Criteria

1. WHEN a convener requests their collection links, THE API SHALL return all collection links belonging to that convener, including cohort name, programme name, token, expiry date, auto_approve setting, and submission count.
2. WHEN a convener updates the auto_approve setting on a collection link, THE Collection_Link_Service SHALL persist the change and apply it to all subsequent submissions through that link.
3. WHEN a convener updates the expiry date on a collection link, THE Collection_Link_Service SHALL persist the updated expiry and enforce it on subsequent submissions.
4. WHEN a convener deletes a collection link, THE Collection_Link_Service SHALL mark the token as revoked so that subsequent submission attempts via that token are rejected.

### Requirement 3: Validate a Collection Link

**User Story:** As a learner, I want to visit a collection link and have the system verify I am eligible to submit a testimonial, so that only genuine cohort participants can leave reviews.

#### Acceptance Criteria

1. WHEN a learner visits a Collection_Link URL, THE Submission_Form SHALL display the programme name and cohort name associated with the token.
2. WHEN the Collection_Token does not exist or has been revoked, THE API SHALL return a 404 response and THE Submission_Form SHALL display an appropriate error message.
3. WHEN the Collection_Token has passed its expiry date, THE API SHALL return a 410 Gone response and THE Submission_Form SHALL inform the learner the link has expired.
4. WHEN a learner submits via a Collection_Link and is not enrolled in the associated cohort, THE API SHALL return a 403 Forbidden response and THE Submission_Form SHALL display an access-denied message.
5. WHEN a learner submits via a Collection_Link and has already submitted a testimonial through that same token, THE API SHALL return a 409 Conflict response and THE Submission_Form SHALL inform the learner they have already submitted.

### Requirement 4: Submit a Testimonial via Collection Link

**User Story:** As a learner, I want to fill in a testimonial form and submit my review, so that my experience is captured and can appear on the convener's organisation page.

#### Acceptance Criteria

1. WHEN a learner submits a testimonial via a valid Collection_Link, THE Submission_Form SHALL collect a quote (required), a rating from 1 to 5 (required), and an optional learner display name override.
2. WHEN a learner submits a valid testimonial and the collection link has auto_approve enabled, THE Collection_Link_Service SHALL create a testimonial record with `is_featured = true`.
3. WHEN a learner submits a valid testimonial and the collection link has auto_approve disabled, THE Collection_Link_Service SHALL create a testimonial record with `is_featured = false`.
4. WHEN a testimonial is created via a Collection_Link, THE Collection_Link_Service SHALL populate `learner_name` from the learner's profile name and `programme_name` from the cohort's associated programme.
5. WHEN a learner submits a testimonial with a quote shorter than 10 characters, THE API SHALL return a 400 Bad Request response.
6. WHEN a learner submits a testimonial with a rating outside the range 1–5, THE API SHALL return a 400 Bad Request response.
7. WHEN a testimonial is successfully submitted, THE Submission_Form SHALL display a confirmation message to the learner.

### Requirement 5: Convener UI for Collection Links

**User Story:** As a convener, I want a dedicated UI section within my cohort management pages to generate and share collection links, so that the workflow is integrated into my existing tools.

#### Acceptance Criteria

1. WHEN a convener views a cohort detail page, THE Convener_UI SHALL display a "Testimonial Collection Link" section showing the current link (if any), its status (active, expired, revoked), and submission count.
2. WHEN a convener clicks "Generate Link", THE Convener_UI SHALL call the API and display the resulting shareable URL with a copy-to-clipboard button.
3. WHEN a convener clicks "Regenerate Link", THE Convener_UI SHALL prompt for confirmation before invalidating the existing token and generating a new one.
4. WHEN a convener clicks "Revoke Link", THE Convener_UI SHALL prompt for confirmation before revoking the token.
5. WHEN a convener toggles the auto_approve setting, THE Convener_UI SHALL immediately persist the change via the API and reflect the updated state.
